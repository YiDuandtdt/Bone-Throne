# Phase 16.2 - Combat Impact Timing Fix

日期：2026-06-06

## 目标

修正攻击演出顺序，让战斗表现更符合预期：

- 攻击方先播放攻击/施法动作
- 命中时点再让防守方受击
- 受击同时刷新血量变化

## 修改内容

### `CombatSystem`

- 普攻不再在触发攻击动画的同一帧立刻结算命中伤害
- 新增一个轻量延迟后命中结算协程
- 攻击者仍然立即消耗行动，避免改变现有回合规则

### `SkillSystem`

- 技能不再先扣血再播放施法动作
- 改为先播放施法动画，再在短延迟后执行 `SkillEffectExecutor` / `DamageResolver`
- 技能冷却和行动消耗仍然在技能确认时立即生效

## 验证重点

1. 普攻时，先看到攻击方挥击，再看到目标受击与掉血。
2. 技能时，先看到施法方动作，再看到目标受击与掉血。
3. 敌方回合攻击时，顺序也保持一致。
4. 击杀目标后，死亡动画和血量归零仍然成立。

## 风险

- 命中结算改为延迟后，若多个动作极短时间叠加，日志显示会略晚于按钮点击
- 若后续攻击动画长度继续调整，可能还要微调 `basicAttackImpactDelay` 和 `skillImpactDelay`
