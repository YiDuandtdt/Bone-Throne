# Phase 16.15 - Frontend Flow And Gameplay Settings Refresh

Date: 2026-06-08

## Goal

Implement a new front-end flow and in-battle settings path without rewriting the three gameplay scenes:

- main menu with `开始游戏 / 继续游戏 / 关于我们 / 设置 / 退出游戏`
- intro story scene with typewriter text and black cover fade
- level-select progression unlocks
- menu settings page and gameplay settings page split
- square action-button layout in battle HUD

## Files Changed

- `Assets/_BoneThrone/Scripts/UI/StartMenuController.cs`
- `Assets/_BoneThrone/Scripts/Core/MenuProgressionState.cs`
- `Assets/_BoneThrone/Scripts/Audio/BTAudioService.cs`
- `Assets/_BoneThrone/Scripts/UI/BattleHUDController.cs`
- `Assets/_BoneThrone/Scripts/UI/SkillBarView.cs`
- `Assets/_BoneThrone/Scenes/IntroStory.unity`
- `ProjectSettings/EditorBuildSettings.asset`

## What Changed

- `StartMenuController` now acts as a shared front-end controller.
  - `StartMenu.unity` uses the default `MainMenu` mode.
  - `IntroStory.unity` overrides the same prefab instance to `IntroStory` mode.
- The menu now builds four runtime pages:
  - Home
  - About
  - Level Select
  - Settings
- `MenuProgressionState` stores:
  - intro completion
  - highest unlocked level
- `BTAudioService` now exposes persistent user volume:
  - `GetBgmVolume / SetBgmVolume`
  - `GetSfxVolume / SetSfxVolume`
- `BattleHUDController` now:
  - registers visited gameplay scenes to unlock level buttons later
  - creates a top-right gameplay settings button and settings page 2
  - keeps `返回游戏` instead of `返回主页`
  - repositions the prompt above the larger action area
- `SkillBarView` now re-lays out the bottom action buttons into a square grid at runtime.

## Inspector / Asset Notes

- `StartMenu.unity` still uses `UI_StartMenuCanvas`; new menu pages are generated by `StartMenuController`.
- `IntroStory.unity` is a copy of `StartMenu.unity` but the prefab instance overrides `sceneMode = IntroStory`.
- Optional customization fields now exist on `StartMenuController` and `BattleHUDController` for:
  - page background sprites
  - page art sprites
  - button sprites
  - button size / position
  - settings panel art
  - intro story text and timing
- `SkillBarView` now has runtime layout fields for:
  - column count
  - square button size
  - spacing
  - padding

## Play Mode Validation

1. Open `Assets/_BoneThrone/Scenes/StartMenu.unity`.
2. Enter Play Mode.
3. Confirm the home page shows five buttons.
4. Click `设置`; confirm settings page 1 appears with:
   - music slider
   - sfx slider
   - return home
   - quit
5. Click `关于我们`; confirm the about page appears and the common top-right settings button opens settings page 1.
6. Click `继续游戏`; confirm the level-select page appears.
7. Confirm level buttons start locked before a new game is completed.
8. Click `开始游戏`; confirm Unity loads `IntroStory`.
9. Confirm:
   - story text appears with a typewriter effect
   - the black cover fades during the middle of the text
   - after the story and a 2 second pause, Unity loads `Level_1`
10. Back in menu after reaching gameplay once, confirm `关卡 1` is unlocked in the level-select page.
11. Reach `Level_2` and `Level_3` normally; then return to menu and confirm their buttons are unlocked.
12. In any gameplay scene, confirm a top-right settings button appears.
13. Open gameplay settings and confirm:
   - music slider
   - sfx slider
   - return game
   - quit
14. Confirm the bottom action buttons are square and re-arranged in a grid.

## Expected Result

- Main menu flow, intro story, level select unlocks, and gameplay settings all work without editing `Level_1/2/3` scene content directly.

## Risks

- The menu and intro UI are generated at runtime by controller code rather than being fully hand-authored scene hierarchies.
- If someone later manually re-enables the old legacy `StartMenu` prefab children, they may overlap the generated pages.
- Gameplay settings overlay is created at runtime, so art replacement is driven by controller fields rather than pre-existing scene children.
